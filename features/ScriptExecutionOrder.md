# Script execution order
This document tracks similarities to Unity execution methods and implementation status.

Glossary:
  - deferred proc - runs later during main loop, usually easy to implement — iterate over components
  - immediate proc - runs immediately, usually harder to implement — requires to be called in all expected places for consistency
  - considered - might be implemented
  - implemented - ready to use

Reference: [Unity 6.3 LTS — Event function execution order](https://docs.unity3d.com/6000.3/Documentation/Manual/execution-order.html).


- Rendering callbacks below apply to the **Built-in Render Pipeline** only; URP/HDRP use different injection points.
- Coroutine vs async relative ordering is not fully guaranteed where the manual calls that out.
