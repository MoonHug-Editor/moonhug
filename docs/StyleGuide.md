# Style Guide

Style Guide for better code consistency, readability, and maintainability

> when not specified "rule of thumb" is to refer to [Odin's Naming & Style Convention](https://github.com/odin-lang/examples/wiki/Naming-and-style-convention)

## Naming
|                 | Case                                                     |
|-----------------|----------------------------------------------------------|
| Import Name     | snake_case (but prefer single word)                      |
| Types           | PascalCase_UnderscoreTolerant                            |
| Enums           | PascalCase_UnderscoreTolerant                            |
| Procedures      | snake_case(but may contain Types with unmodified casing) |
| Local Variables | snake_case                                               |
| Constants       | SCREAMING_SNAKE_CASE                                     |
